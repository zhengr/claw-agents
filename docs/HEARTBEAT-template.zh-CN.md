# HEARTBEAT.md

当智能体收到心跳轮询时，会读取本文件。**保持本文件为空或仅含注释即表示不执行任何动作** — 智能体会回复 `HEARTBEAT_OK`。

当你希望智能体**周期性地检查某些事项**（例如每天 2～4 次）时，在下方添加任务。列表宜短，以控制 token 消耗。

---

## 心跳 vs Cron

| 用 **心跳** 时 | 用 **cron** 时 |
|----------------|-----------------|
| 多项检查可在一轮中一起做（如收件箱 + 日历 + 提醒） | 需要精确时间（如「每周一 9:00」） |
| 希望结合最近对话上下文 | 任务需与主会话隔离执行 |
| 时间可以略有漂移（如约每 30 分钟） | 一次性或固定节奏 |
| 想通过合并检查减少 API 调用 | 需用不同模型或独立进程执行 |

---

## 在下方可添加的内容

- **提醒类** — 如「检查明日会议；若有需要准备的会议则提醒。」
- **周期检查** — 如「检查收件箱未读紧急邮件；若有则摘要。」
- **轻度维护** — 如「浏览近期 `memory/YYYY-MM-DD.md`；若有重要内容则更新 `MEMORY.md`。」

**可选：** 在 `memory/heartbeat-state.json` 中记录上次执行时间，避免重复劳动（如 `{"lastChecks":{"calendar":1703275200}}`）。

**当无需执行任何任务时：** 智能体应回复 `HEARTBEAT_OK`。未经用户同意，不要以心跳名义发消息或执行对外操作。
