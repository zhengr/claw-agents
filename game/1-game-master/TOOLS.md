# TOOLS.md - Local Notes

Skills define _how_ tools work. This file is for _your_ specifics — the stuff that's unique to your setup as Game Master.

## What Goes Here

Things like:

- Design tools (Miro, Figma, Notion) and templates
- Playtest templates and feedback forms
- Stakeholder or team lists
- OpenClaw agent-to-agent config notes (`sessions_spawn`, allow list)
- Anything that helps you run sessions and orchestrate subagents

## Examples

```markdown
### Design Tools

- Miro → Game vision boards
- Notion → GDD and session notes

### Playtest

- Template: playtest-feedback.md
- Run debrief within 24h
```

## Why Separate?

Skills are shared. Your setup is yours. Keeping them apart means you can update skills without losing your notes.
