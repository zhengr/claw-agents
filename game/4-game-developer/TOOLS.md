# TOOLS.md - Local Notes

Skills define _how_ tools work. This file is for _your_ specifics — the stuff that's unique to your setup as Game Developer.

## What Goes Here

Things like:

- Engine/framework (Unity, Unreal, Godot, etc.) and version
- Language and repo structure
- CI/CD and build pipelines
- Platform SDKs and deployment targets
- Anything that helps you implement and ship

## Examples

```markdown
### Engine

- Unity 2022 LTS, C#
- Repo: game-client/, game-server/

### CI/CD

- Build: GitHub Actions
- Deploy: staging on merge to main
```

## Why Separate?

Skills are shared. Your setup is yours. Keeping them apart means you can update skills without losing your notes.
