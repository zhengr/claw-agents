# TOOLS.md - Local Notes

Skills define _how_ tools work. This file is for _your_ specifics — the stuff that's unique to your setup.

## What Goes Here

Things like:

- Doc paths, API base URLs, environment names
- SSH hosts and aliases
- Preferred formatting for code or commands
- Anything environment-specific for tech support

## Examples

```markdown
### Docs / API

- API docs: https://...
- FAQ path: /path/to/tech-faq

### SSH

- staging → 192.168.1.x, user: deploy
```

## Why Separate?

Skills are shared. Your setup is yours. Keeping them apart means you can update skills without losing your notes.
