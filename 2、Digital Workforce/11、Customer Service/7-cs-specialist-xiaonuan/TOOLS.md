# TOOLS.md - Local Notes

Skills define _how_ tools work. This file is for _your_ specifics — the stuff that's unique to your setup.

## What Goes Here

Things like:

- Script library path or knowledge-base ID (by scenario: presale/aftersale/complaint/consultation); external replies use only these scripts
- Company business content path (product, policy, FAQ, brand tone)
- Industry info path or source
- Escalation and handoff rules (when to hand to human, ticket entry) — can go here or in MEMORY.md

## Examples

```markdown
### Scripts

- Presale: path or KB id
- Aftersale: path or KB id
- Complaint: path or KB id

### Business

- Product/Policy/FAQ: path or link
- Brand tone: (short note)

### Escalation

- Hand to human: (rules)
- Ticket entry: (link or instructions)
```

## Why Separate?

Skills are shared. Your setup is yours. Update this when scripts and knowledge change so Xiaonuan's replies stay evidence-based.
